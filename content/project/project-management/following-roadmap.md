title: Following the roadmap: delivering subprojects
what: >
  The roadmap should be divided into a set of distinct features or deliverables that can be planned separately and worked on independently
why: >
  Dividing the project into subprojects provides a framework for planning work, managing scope, setting priorities, estimating effort, and assigning work
when: Annually
implementations:
  - desc: See [Annual roadmaps]()
  - desc: Map out dependencies between subprojects to adjust priority and set sequence
  - desc: Set a delivery date for each subproject based on priority (higher priority things first) and dependencies, assign appropriate team members, estimate the length of calendar time required to complete it with given assignments, and then schedule work on a quarterly basis
